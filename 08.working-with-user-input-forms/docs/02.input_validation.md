# Input Validation & Formatters

## Validators

A validator is a plain Dart function with the signature `String? Function(String? value)`. It returns `null` when the value is acceptable, or a non-empty error string when it is not. The `Form` widget calls each field's validator when `FormState.validate()` is invoked.

### Basic Pattern

```dart
String? _nameValidator(String? value) {
  final trimmed = (value ?? '').trim();
  if (trimmed.isEmpty) {
    return 'Product name is required';
  }
  if(trimmed.length < 5){
    return 'Product name must be at least 5 characters long';
  }
  return null;
}

TextFormField(
  controller: nameController,
  decoration: const InputDecoration(hintText: 'Enter product name'),
  validator: _nameValidator,
)
```

The validator is attached directly to the `TextFormField` via the `validator` parameter. When `_formKey.currentState?.validate()` is called, Flutter calls this function and, if it returns a non-null string, displays it as an error message below the field using the theme's `errorBorder` styling.

### Numeric Validation

```dart
String? _priceValidator(String? value) {
  if (value == null || value.trim().isEmpty) {
    return 'Price is required';
  }
  if ((double.tryParse(value) ?? 0) <= 0) {
    return 'Invalid price';
  }
  return null;
}
```

`double.tryParse` returns `null` if the string cannot be parsed as a number. The `?? 0` fallback safely converts that `null` into `0`, which then fails the `> 0` check, producing the error message.

### Optional Numeric Fields

Some fields are optional but still have constraints when a value is provided. The validator should only run the range check when the field is not empty:

```dart
String? _discountValidator(String? value) {
  if (value != null && value.isNotEmpty) {
    final val = double.tryParse(value);
    if (val == null || val < 0 || val > 100) {
      return 'Must be between 0-100';
    }
  }
  return null; // Empty = valid (field is optional)
}
```

### URL Validation

`Uri.tryParse` returns `null` when the input cannot be parsed as a URI, making it a convenient first step for validating URL input. In many apps, it's also worth checking that the parsed URI uses an expected scheme (such as `http` or `https`) and includes a host.

```dart
String? _validator(String? value) {
  if (value == null || value.trim().isEmpty) {
    return 'Image URL is required';
  }
  if (Uri.tryParse(value) == null) {
    return 'Invalid image URL';
  }
  return null;
}
```

## FieldLabel Widget

The project defines a reusable `FieldLabel` widget to consistently display labels above form fields. It uses `RichText` to optionally render a red asterisk for required fields without repeating that styling logic in every field widget:

```dart
// features/my_products/presentation/widgets/field_label.dart
class FieldLabel extends StatelessWidget {
  const FieldLabel({super.key, required this.label, this.isRequired = false});

  final String label;
  final bool isRequired;

  @override
  Widget build(BuildContext context) {
    return Padding(
      padding: const EdgeInsets.only(bottom: AppDimensions.defaultMargin / 4),
      child: RichText(
        text: TextSpan(
          text: label,
          style: Theme.of(context).textTheme.bodySmall?.copyWith(
            color: Theme.of(context).colorScheme.onSurface,
          ),
          children: [
            if (isRequired)
              TextSpan(
                text: ' *',
                style: Theme.of(context).textTheme.bodySmall?.copyWith(
                  color: Theme.of(context).colorScheme.secondary,
                ),
              ),
          ],
        ),
      ),
    );
  }
}
```

Every form field widget in the `my_products` feature follows the same structure: a `FieldLabel` above the `TextFormField`, grouped in a `Column`:

```dart
Column(
  crossAxisAlignment: CrossAxisAlignment.start,
  children: [
    const FieldLabel(label: 'Product name', isRequired: true),
    TextFormField(
      controller: nameController,
      validator: _nameValidator,
    ),
  ],
)
```

## Input Formatters

Validators fire only when the form is submitted. To restrict what characters can be typed in the first place (before submission) - use `inputFormatters`. These run on every keystroke and silently filter out any characters that don't match the allowed pattern.

The project defines `AppInputFormatters` as a utility class in `shared/utils/input_formatters.dart`:

```dart
import 'package:flutter/services.dart'
    show FilteringTextInputFormatter, TextInputFormatter;

class AppInputFormatters {
  static TextInputFormatter decimal({
    required int decimalPlaces,
    bool allowNegative = false,
  }) {
    final signPattern = allowNegative ? r'-?' : '';
    return FilteringTextInputFormatter.allow(
      RegExp('^$signPattern\\d*\\.?\\d{0,$decimalPlaces}'),
    );
  }

  static final digitsOnly = FilteringTextInputFormatter.digitsOnly;
}
```

Two formatters are exposed:

| Formatter                                      | What it allows                                               |
| :--------------------------------------------- | :----------------------------------------------------------- |
| `AppInputFormatters.decimal(decimalPlaces: 2)` | Digits and up to 2 decimal places (`0.00`, `12.5`, `999.99`) |
| `AppInputFormatters.digitsOnly`                | Integers only, no decimal point                              |

They are applied to `TextFormField` via the `inputFormatters` list:

```dart
// Price field: allow decimals with up to 2 places
TextFormField(
  controller: priceController,
  keyboardType: const TextInputType.numberWithOptions(decimal: true),
  inputFormatters: [AppInputFormatters.decimal(decimalPlaces: 2)],
  validator: _priceValidator,
)

// Discount field: whole numbers only
TextFormField(
  controller: discountController,
  keyboardType: TextInputType.number,
  inputFormatters: [AppInputFormatters.digitsOnly],
  validator: _discountValidator,
)
```

> **Note:** Input formatters and validators are complementary, not redundant. Formatters prevent invalid input as it is typed. Validators confirm the complete submitted value is within the expected business constraints.

## Debounced Live Preview

Some fields need to respond to user input as the user types, but not after every single keystroke. Debouncing delays updates until the user pauses typing, reducing unnecessary work and improving performance. The `ProductImageField` shows a live image preview as the user types a URL, but waits 2s after each keystroke before updating:

```dart
class _ProductImageFieldState extends State<ProductImageField> {
  Timer? _debounceTimer;
  bool hasValidImageUrl = false;

  @override
  void initState() {
    super.initState();
    widget.imageUrlController.addListener(_onImageUrlChanged);
  }

  @override
  void dispose() {
    _debounceTimer?.cancel();
    widget.imageUrlController.removeListener(_onImageUrlChanged);
    super.dispose();
  }

  void _onImageUrlChanged() {
    _debounceTimer?.cancel();
    _debounceTimer = Timer(const Duration(milliseconds: 2000), () {
      if (!mounted) return;
      final uri = Uri.tryParse(widget.imageUrlController.text);
      setState(() {
        hasValidImageUrl = uri != null;
      });
    });
  }
}
```

Key details:

- A listener is added in `initState` and **removed in `dispose`**. Skipping removal causes the listener to run against a disposed widget, leading to crashes.
- `_debounceTimer?.cancel()` on every keystroke ensures only the last timer fires. If the user types quickly, intermediate timers are discarded.
- The `mounted` check guards against state updates after the widget has been disposed.

This pattern is a direct application of the debouncing technique described in [Architecture and State Management Chapter](../../07.architecture-and-state-management/docs/04.advanced_patterns.md#debouncing-user-input).

## Custom Dropdown Widget (AppDropdown)

Flutter's built-in `DropdownButton` uses a `PopupRoute` that is difficult to style to match the rest of the form. The project provides `AppDropdown`, a custom stateful widget built from `GestureDetector`, `AnimatedCrossFade`, and `AnimatedRotation` that opens and closes inline below the trigger:

```dart
class AppDropdown extends StatefulWidget {
  const AppDropdown({
    super.key,
    required this.value,
    required this.items,
    required this.onChanged,
  });

  final String value;
  final List<String> items;
  final ValueChanged<String> onChanged;
}
```

The open/closed state is entirely **ephemeral** and lives inside `_AppDropdownState._isOpen`. It does not need to be in a Riverpod provider because no other widget needs to know whether the dropdown is open.

Animated transitions provide visual feedback without any explicit animation controller:

- `AnimatedCrossFade` toggles between `SizedBox.shrink()` (closed) and the options list (open) over 200ms.
- `AnimatedRotation` rotates the chevron icon 180° when open, smoothly indicating the direction the list will close.

The dropdown filters the current value from the options list so the selected item does not appear again as a choice:

```dart
List<String> get _filteredItems {
  return widget.items.where((item) => item != widget.value).toList();
}
```
