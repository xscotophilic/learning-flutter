# The `my_store` Showcase App

This chapter extends the `my_store` app from [Architecture and State Management Chapter](../../07.architecture-and-state-management/docs/05.my_store_walkthrough.md) with a full product management feature. Users can now create, edit, and delete their own products through a form-driven interface.

## What Was Added

### New Feature: `my_products`

A new `my_products` feature module was added under `features/`. It follows the same three-layer clean architecture established in previous chapter:

```
features/my_products/
├── domain/
│   └── entities/
│       └── my_products.dart
└── presentation/
    ├── pages/
    │   ├── my_products_page.dart                # List of the current user's products
    │   └── product_form_page.dart               # Add / Edit form
    ├── providers/
    │   ├── my_products_notifier.dart            # CRUD state coordinator
    │   ├── my_products_notifier.g.dart
    │   ├── my_products_usecase_providers.dart   # Use case wiring
    │   └── my_products_usecase_providers.g.dart
    └── widgets/
        ├── field_label.dart                     # Reusable label
        ├── product_currency_field.dart
        ├── product_description_field.dart
        ├── product_image_field.dart
        ├── product_name_field.dart
        └── product_price_and_discount_field.dart
```

### Extended: `product` Feature

Four new use cases were added under `features/product/domain/usecases/`:

- `GetMyProductsUseCase`: fetches the current user's products and returns only their IDs
- `CreateProductUseCase`: delegates product creation to the repository
- `UpdateProductUseCase`: delegates product update to the repository
- `DeleteProductUseCase`: delegates product deletion to the repository

`ProductRepository` was extended with the four corresponding abstract methods. `MockProductRepository` implements them by delegating to the new `MockServer` static methods.

`Product` and `ProductModel` were both updated to carry a `creatorId` field, which the mock server uses to scope product ownership.

### Extended: `MockServer`

Four new static methods were added to `MockServer`:

- `getMyProducts`: filters `MockProductsData.products` by `Authorization` header token
- `createProduct`: appends a new entry with a random ID to `MockProductsData.products`
- `updateProduct`: patches an existing entry, verifying ownership before writing
- `deleteProduct`: removes an entry by ID, verifying ownership before deleting

`MockProductsData.products` was changed from `static const` to `static List` (mutable), and a `creator_id` field was added to each seed record, so the mock server can filter and enforce ownership checks.

### Extended: Routing & Navigation

`AppRoutes` was updated with two new cases:

- `/my-products` -> `MyProductsPage`
- `/product-form` -> `ProductFormPage(productId: ...)`, where `productId` is passed as a route `arguments` string

The `AppDrawer` now includes a "My Products" entry that pushes `/my-products`.

### Extended: Shared Widgets & Utilities

- `AppDropdown`: a custom animated dropdown widget (see [section 02](02.input_validation.md#custom-dropdown-widget-appdropdown))
- `AppInputFormatters`: decimal and digits-only formatters (see [section 02](02.input_validation.md#input-formatters))

## Local Setup

To run the application locally on your machine or emulator, follow these steps:

```bash
cd my_store
flutter pub get
dart run build_runner build --delete-conflicting-outputs  # regenerate .g.dart files
flutter run
```
